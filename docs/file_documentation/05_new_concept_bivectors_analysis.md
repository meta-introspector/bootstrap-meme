# 05_new_concept_bivectors.md - Conceptual Multivector Analysis

## Summary
This document introduces the concept of "Bivectorial Relationships" within System `e`, utilizing the bivector component of a term's multivector embedding. The bivector, representing an oriented plane, will be used to encode the primary relational plane between two concepts. The magnitude of this bivector will signify the strength of that relationship, as exemplified by terms like "SOP" and "Git."

## Relationships between Concepts (Conceptual Geometric Operations)

*   **Bivectorial Relationships** *are introduced through* the **Bivector component** of a **Multivector embedding**. (Conceptual "introduction" or "application")
    *   `Bivectorial Relationships` via `Bivector component` of `Multivector embedding`
*   The **Bivector component** *represents* an **Oriented plane**. (Conceptual "representation")
    *   `Bivector component` represents `Oriented plane`
*   In **System `e`**, the **Oriented plane** *will be used to* **Encode primary relational plane** between **Two concepts**. (Conceptual "purpose" or "function")
    *   `System e` uses `Oriented plane` to `Encode primary relational plane` between `Two concepts`
*   The **Strength of the relationship** *is represented by* the **Magnitude of bivector**. (Conceptual "quantification")
    *   `Strength of the relationship` = `Magnitude of bivector`
